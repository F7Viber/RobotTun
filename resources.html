Robot Runner: Student Guide

Welcome! This guide will help you get set up and start building your own endless runner game.

Getting Started with GitHub Pages

Follow these steps to set up your workspace and preview your game live:

  1. 1Fork the repository
    Go to F7Viber/RobotTun and click the "Fork" button in the top right to create your own copy.
  2. 2Clone your fork
    Open your terminal and run:
    git clone https://github.com/YOUR-USERNAME/RobotTun.git
    (Replace YOUR-USERNAME with your GitHub username)
  3. 3Open the project
    Open the project folder in VS Code or your favorite editor.
  4. 4Start a local server
    Open the terminal and run:
    python3 -m http.server 8000
    Then open http://127.0.0.1:8000/ in your browser.
  5. 5Edit and refresh
    Edit index.html in your editor, save, then refresh your browser to see changes.
  6. 6Push to GitHub Pages
    When you're ready to share, commit your changes and push them:
    git add .
    git commit -m "Initial endless runner game"
    git push origin main
    Your game will be live at: https://YOUR-USERNAME.github.io/RobotTun/

Your First Challenge

Create an endless runner game where:

  • A robot character appears on screen and can jump when you press the spacebar
  • Obstacles spawn from the right side and move toward the left
  • If the robot collides with an obstacle, the game ends and shows your score
  • As time passes, the game gets faster
  • Display the current score on screen

Start with:

  • Draw simple shapes (rectangles) for the robot and obstacles using the Canvas API
  • Implement keyboard input to make the robot jump
  • Move obstacles across the screen in a game loop
  • Check if the robot and obstacle overlap (collision detection)

Tips for Success

  • Start simple: Get the basic game loop working before adding fancy graphics.
  • Test often: Refresh your browser after every few changes.
  • Break it down: Work on jumping, then obstacles, then collisions, then scoring.
  • Use the console: Press F12 to open developer tools and check for errors.
  • Ask for help: If you're stuck, describe what you're trying to do and what's happening instead.

Next Steps (Build on Your Game)

Once you have the basics working, try adding:

  • Better graphics and animations
  • Sound effects
  • Different obstacle types
  • Power-ups or special items
  • A leaderboard to track high scores
